# 维护巡检清单

本文件记录 `astrbot_plugin_points_mall` 的轻量维护与上线前检查项，便于后续巡逻和人工接手时快速确认风险点。

## 每轮巡检建议

1. **仓库结构**
   - 确认插件入口文件 `main.py`、`metadata.yaml`、`_conf_schema.json` 存在。
   - 避免继续提交运行时缓存目录，例如 `__pycache__/`。
   - README 中的命令、配置项与实际代码保持同步。

2. **配置与权限**
   - 检查 `super_admins` 和 `admins` 是否只包含必要账号。
   - 敏感库存、固定发货内容、库存导出等命令应保持私聊限制。
   - 不在仓库中提交真实卡密、兑换码、群号、用户 QQ 列表、token 或 cookie。

3. **数据安全**
   - 维护前备份 `data/plugins/astrbot_plugin_points_mall/`。
   - 重点关注 `points_data.json`、库存分片、订单分片和 `.bak` 文件。
   - 变更保存逻辑时，确认临时文件和原子替换流程仍可回滚。

4. **功能回归**
   - `/签到`：同日重复签到应被拦截。
   - `/积分商场`：只展示已上架商品，库存数量展示不泄露真实内容。
   - `/购买 [序号]`：必须私聊执行，余额不足、库存不足、商品不存在均应给出明确提示。
   - `/商城自检`：能发现重复 ID、异常订单、库存状态和已售内容删除流程问题。

5. **库存与隐私**
   - 固定内容发货应记录版本和哈希，不在公开回复中暴露真实内容。
   - 一次性库存售出后按 `sold_stock_retention_days` 清理真实内容。
   - 导出库存和查看库存明细时保持脱敏或私聊约束。

## 上线前最小校验

```bash
python -m py_compile main.py
python -m json.tool _conf_schema.json >/dev/null
```

如本地具备 AstrBot 测试环境，建议额外执行一次插件加载、私聊购买、强制退款和商城自检流程。

## 回滚建议

- 文档变更：直接 revert 对应 commit 即可。
- 代码变更：先停用插件或回退到上一版本，再恢复维护前备份的数据目录。
- 数据异常：优先使用 `.bak` 与巡检前备份恢复，恢复后执行 `/商城自检`。
