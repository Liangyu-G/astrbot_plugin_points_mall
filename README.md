# AstrBot 积分商城插件

`astrbot_plugin_points_mall` 是一个基于 AstrBot 的积分商城插件，支持签到积分、群聊活跃积分、商品上架审核、自动发货、订单记录、强制退款、库存管理、低库存提醒、积分流水和数据分片存储。

## 功能概览

### 积分获取

- 每日签到：用户每天可通过 `/签到` 获取随机积分。
- 群聊活跃奖励：群聊文字消息长度必须**严格超过**配置阈值才会奖励积分。
- 活跃冷却：同一用户两次活跃奖励之间有冷却时间，防止刷屏刷分。
- 超级管理员可通过 `/给予积分` 增加或扣除用户积分。
- 支持 `/积分流水` 查询指定用户最近积分变动记录。

### 积分商城

- `/积分商场` 展示所有已上架商品。
- 商品按照序号排列。
- 商品价格必须大于 `0`。
- 普通管理员可提交上架申请。
- 超级管理员可直接上架，也可审核普通管理员提交的商品。
- 下架采用软删除，历史订单仍保留。

### 自动发货

插件当前为全自动发货模式，支持两种发货类型：

1. 固定内容发货
   - 使用 `/设置发货 [序号] [固定发货内容]` 配置。
   - 适合网盘链接、兑换说明、教程地址等可重复发放内容。
   - 每次修改会记录发货版本和内容哈希。

2. 一次性库存发货
   - 使用 `/添加库存` 或 `/批量添加库存` 添加卡密、兑换码等一次性内容。
   - 用户购买后自动占用一条未售库存。
   - 支持库存去重。
   - 支持低库存提醒。

### 订单与退款

- 用户必须私聊机器人执行 `/购买 [序号]`。
- 购买成功后自动扣除积分、生成订单并立即发货。
- 订单状态包括：
  - `completed`：已完成
  - `refunded`：已退款
- 管理员可通过 `/订单列表` 和 `/查订单` 查看订单。
- 超级管理员可通过 `/强制退款 [订单号]` 对已完成订单退款。
- 强制退款只退积分，不自动回收已发出的卡密或内容。

### 已售库存内容删除流程

库存型商品售出后，真实库存内容会进入删除流程。

- 默认保留 `30` 天。
- `0` 表示售出后下次保存或清理时立即删除真实内容。
- `-1` 表示不自动删除。
- 到期后只清空真实 `content`，保留库存元数据和订单关联信息。
- 保留字段包括库存 ID、买家、订单号、售出时间、删除时间、内容哈希等。
- 可通过 `/清理到期已售库存` 手动触发清理。
- 可通过 `/设置已售保留天数 [天数]` 修改保留策略。

### 安全与隐私

- 购买必须私聊。
- 真实发货内容强制禁用 T2I，避免卡密被转成图片。
- 以下敏感命令必须私聊使用：
  - `/设置发货`
  - `/添加库存`
  - `/批量添加库存`
  - `/查看库存`
  - `/库存明细`
  - `/导出库存`
  - `/清空库存`
  - `/清理已售库存`
- 管理员通知通过私聊发送给超级管理员和商品上架人。

### 数据可靠性

- 商品、订单和库存支持 500 条分片存储。
- 保存数据时使用临时文件和原子替换。
- 保存前会保留 `.bak` 备份。
- 购买流程使用并发锁，避免并发购买同一库存。
- 提供 `/商城自检` 检查数据健康状态。

## 安装方法

1. 将插件目录放入 AstrBot 的 `data/plugins/` 目录。
2. 重启 AstrBot，或在插件管理页重新加载插件。
3. 在 AstrBot 插件配置页中配置超级管理员和普通管理员 QQ。
4. 发送 `/积分帮助` 查看全部指令。

## 配置项

配置文件为 `_conf_schema.json`，也可以在 AstrBot 管理面板中修改。

| 配置项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `super_admins` | `list` | `[]` | 超级管理员 QQ 列表，拥有授权上架、强制退款、商城自检、设置已售库存删除周期和调整积分等权限。 |
| `admins` | `list` | `[]` | 普通管理员 QQ 列表，可提交商品上架申请，并管理自己上架商品的自动发货库存。 |
| `sign_min` | `int` | `10` | 每日签到最小积分。 |
| `sign_max` | `int` | `50` | 每日签到最大积分。 |
| `active_text_len` | `int` | `10` | 活跃文字字数阈值，群聊消息文字数必须严格超过该值才奖励积分。默认 `10` 表示至少 `11` 个字。 |
| `active_point_reward` | `int` | `1` | 单次活跃奖励积分。 |
| `active_cooldown` | `int` | `60` | 活跃积分冷却时间，单位秒。 |
| `low_stock_warn_threshold` | `int` | `3` | 库存型商品购买后，剩余库存小于或等于该值时提醒超级管理员和上架人。 |
| `sold_stock_retention_days` | `int` | `30` | 已售库存真实内容保留天数。`0` 表示立即删除，`-1` 表示不自动删除。 |
| `use_t2i` | `bool` | `true` | 是否让普通回复接入 AstrBot T2I。真实发货内容会强制禁用 T2I。 |

> 说明：`sold_stock_retention_days_manual` 是运行时内部标记，会写入 `points_data.json`，不是配置项，不需要写入 `_conf_schema.json`。

| 指令 | 说明 |
| --- | --- |
| `/积分帮助` | 查看插件帮助。 |
| `/签到` | 每日签到获取随机积分。 |
| `/我的积分` | 查询自己的积分余额。 |
| `/积分排行` | 查看积分排行榜前 10 名。 |

| `/购买 [序号]` | 私聊购买商品并自动发货。 |

### 管理员指令

| 指令 | 说明 |
| --- | --- |
| `/上架 [名称] [价格] [描述]` | 上架或申请上架商品，价格必须大于 `0`。 |
| `/设置发货 [序号] [固定发货内容]` | 私聊设置固定内容自动发货。 |
| `/添加库存 [序号] [卡密/兑换码]` | 私聊添加一条一次性库存。 |
| `/批量添加库存 [序号] [多行库存或 \| 分隔库存]` | 私聊批量添加一次性库存。 |
| `/查看库存 [序号]` | 私聊查看商品库存数量。 |
| `/库存明细 [序号]` | 私聊查看库存脱敏明细和已售删除状态。 |
| `/导出库存 [序号]` | 私聊导出未售库存真实内容。 |
| `/清空库存 [序号]` | 私聊清空未售库存，保留已售记录。 |
| `/清理已售库存 [序号] [天数]` | 私聊手动强制清理指定天数前的已售库存记录。 |
| `/待审核商品` | 查看待审核商品。 |
| `/下架 [序号]` | 下架商品。 |
| `/订单列表 [已完成/已退款/全部]` | 查看最近订单。 |

### 超级管理员指令

| 指令 | 说明 |
| --- | --- |
| `/授权上架 [序号]` | 审核通过商品。 |
| `/给予积分 [QQ号/@用户] [积分数量]` | 增加或扣除指定用户积分。 |
| `/强制退款 [订单号]` | 对已完成订单强制退款。 |
| `/商城自检` | 检查商品、订单、库存、删除流程和积分流水状态。 |
| `/积分流水 [QQ号]` | 查看指定用户最近积分变动。 |
| `/设置已售保留天数 [天数]` | 设置已售库存真实内容保留时间，`-1` 表示不自动删除。 |
| `/清理到期已售库存` | 按删除流程清理已到期的已售库存真实内容。 |

```text

/授权上架 1

/清理到期已售库存
```

## 数据文件说明

插件数据默认存放在：

```text
data/plugins/astrbot_plugin_points_mall/
```

主要文件：

| 文件 | 说明 |
| --- | --- |
| `points_data.json` | 主数据文件，保存用户积分、计数器、首批商品和首批订单等。 |
| `points_data_items_2.json`、`points_data_items_3.json` ... | 商品超过 500 条后的分片文件。 |
| `points_data_orders_2.json`、`points_data_orders_3.json` ... | 订单超过 500 条后的分片文件。 |
| `points_stock_item_商品ID_1.json`、`points_stock_item_商品ID_2.json` ... | 商品库存分片文件，每 500 条一片。 |
| `*.bak` | 保存数据前自动生成的备份文件。 |
| `*.tmp` | 原子写入时的临时文件，正常情况下不会长期保留。 |

## 权限规则

| 角色 | 权限 |
| --- | --- |
| 普通用户 | 签到、查积分、看排行、看商城、私聊购买。 |
| 普通管理员 | 申请上架商品，管理自己上架商品的发货内容和库存，查看相关订单。 |
| 超级管理员 | 拥有全部权限，包括授权上架、调整积分、强制退款、商城自检、设置已售内容删除周期。 |

## 运维建议

- 定期备份 `data/plugins/astrbot_plugin_points_mall` 目录。
- 使用 `/商城自检` 检查重复 ID、异常订单、库存状态和删除流程。
- 使用 `/积分流水 [QQ号]` 处理积分争议。
- 使用 `/查订单 [订单号]` 排查订单问题。
- 卡密、兑换码等敏感库存建议只在私聊中维护。

## 注意事项

- 自动发货内容一旦发出，强制退款不会自动回收。
- 固定内容发货适合可重复使用的内容；一次性卡密建议使用库存发货。
- 如果平台不允许机器人主动私聊用户，管理员通知或购买通知可能发送失败，但订单与数据仍会保存。
- 修改配置后建议重载插件或重启 AstrBot，确保配置生效。
