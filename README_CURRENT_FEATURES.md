# 积分商城当前功能说明

## 核心能力

- 群聊活跃积分：消息文字数必须严格超过配置阈值才奖励积分。
- 每日签到积分：支持配置随机积分范围。
- 积分商城：商品按序号展示，群员可私聊购买。
- 自动发货：支持固定内容发货和一次性库存发货。
- 普通管理员上架需超级管理员审核，超级管理员可直接上架。

## 安全规则

- 购买必须私聊。
- 设置发货、添加库存、批量添加库存、导出库存、库存明细、清理库存必须私聊。
- 真实发货内容、订单详情和库存导出强制禁用 T2I。
- 商品下架为软删除，历史订单仍可追溯。

## 数据文件

主数据文件：

- `points_data.json`

超过 500 条后自动分片：

- 商品：`points_data_items_2.json`, `points_data_items_3.json`, ...
- 订单：`points_data_orders_2.json`, `points_data_orders_3.json`, ...
- 库存：`points_stock_item_商品ID_1.json`, `points_stock_item_商品ID_2.json`, ...

保存时会先写入 `.tmp` 临时文件，再原子替换正式 JSON，并保留 `.bak` 备份。

## 常用指令

### 用户

- `/签到`
- `/我的积分`
- `/积分排行`
- `/积分商场`
- `/购买 [序号]`

### 管理员

- `/上架 [名称] [价格] [描述]`
- `/设置发货 [序号] [固定发货内容]`
- `/添加库存 [序号] [卡密/兑换码]`
- `/批量添加库存 [序号] [多行库存或 | 分隔库存]`
- `/查看库存 [序号]`
- `/库存明细 [序号]`
- `/导出库存 [序号]`
- `/清空库存 [序号]`
- `/清理已售库存 [序号] [天数]`
- `/待审核商品`
- `/下架 [序号]`
- `/订单列表 [已完成/已退款/全部]`
- `/查订单 [订单号]`

### 超级管理员

- `/授权上架 [序号]`
- `/给予积分 [QQ号/@用户] [积分数量]`
- `/强制退款 [订单号]`
- `/商城自检`
- `/积分流水 [QQ号]`

## 运维建议

- 定期备份 `data/plugins/astrbot_plugin_points_mall` 目录。
- 使用 `/商城自检` 检查重复 ID、异常订单、库存状态。
- 使用 `/积分流水 [QQ号]` 处理积分争议。
- 使用 `/查订单 [订单号]` 排查订单问题。
